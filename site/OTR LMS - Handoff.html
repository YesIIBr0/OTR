OTR LMS — Tema hijo de Boost

Cómo el prototipo se implementa como un tema de Moodle real: instalación, mapeo de tokens a SCSS de Bootstrap, y cada pantalla conectada a su ruta y su plantilla Mustache con su esquema de contexto.

01Instalación

El tema vive en theme/otr/ dentro de tu Moodle (4.3+). Copia la carpeta, entra como admin y completa la actualización; Moodle detecta el plugin.

# 1. Copia el plugin a la raíz de Moodle
cp -r theme/otr  /var/www/moodle/theme/otr

# 2. Dispara la actualización (UI o CLI)
php admin/cli/upgrade.php --non-interactive

# 3. Activa el tema y purga cachés
# Site administration › Appearance › Themes › Theme selector → "OTR"
php admin/cli/purge_caches.php
Requisito: theme_boost debe estar presente — OTR hereda todos sus layouts y renderers (declarado en version.php → $plugin->dependencies).

02Estructura del plugin

Estructura mínima de un tema hijo de Boost. Cada archivo cumple un rol concreto del pipeline de Moodle.

theme/otr/ ├── version.php # metadata + dependencia de Boost ├── config.php # $THEME: parents=[boost], callbacks SCSS ├── lib.php # theme_otr_get_(main|pre|extra)_scss() ├── settings.php # preset + colores de marca (Appearance) ├── lang/ │ ├── en/theme_otr.php │ └── es/theme_otr.php ├── scss/ │ ├── pre.scss # tokens → variables Bootstrap (ANTES) │ └── post.scss # overrides por componente (DESPUÉS) ├── templates/ │ ├── core/ │ │ └── loginform.mustache # override de /login │ └── theme_otr/ │ └── level_badge.mustache # partial de marca reutilizable └── pix/ # escudo, favicon, fuentes self-host
Regla de oro de los overrides: para sobreescribir cualquier plantilla, replica su ruta de componente dentro de templates/. Ej: core/full_header → theme/otr/templates/core/full_header.mustache; mod_forum/forum_discussion → theme/otr/templates/mod_forum/forum_discussion.mustache. Moodle resuelve la versión del tema automáticamente.

05Esquemas de contexto (ejemplos)

Las plantillas Mustache reciben un contexto JSON. Estos son los campos clave de las que vas a tocar. Respétalos: si quitas un campo del form de login, rompes el login.

core/loginform — ya incluido como override

{
  "error": "",            // mensaje de error
  "info": "",
  "loginurl": ".../login/index.php",
  "username": "",         // usuario recordado
  "rememberusername": true,
  "logintoken": "…",       // CSRF — NO quitar
  "identityproviders": [ { "url": "…", "name": "Google" } ],
  "cansignup": true,
  "instructions": "<p>…</p>"
}

theme_otr/level_badge — partial de marca (nuestro)

{
  "levelid": "varsity",   // novato | jv | varsity | elite
  "label": "Varsity",
  "xp": 3120,
  "compact": false
}
// uso desde cualquier template:
{{> theme_otr/level_badge }}
Para descubrir el esquema de cualquier plantilla: lee el bloque @template al inicio del .mustache de core, o ejecuta el renderer con $OUTPUT->render_from_template('core/loginform', $context) y vuelca $context.

06Niveles & gamificación

La progresión Novato → JV → Varsity → Elite es el ADN de campeón de OTR. En Moodle se implementa con el bloque block_xp (“Level Up!”), reetiquetando sus 4 niveles y aplicando los colores de marca vía post.scss.

NivelRango XPColor · variable
Novato0 – 999$otr-level-novato #8499B2
JV1.000 – 2.499$otr-level-jv #4FA9E8
Varsity2.500 – 4.999$otr-level-varsity #2E8BD0
Elite5.000+$otr-level-elite #0C2340

Renderiza el badge con el partial theme_otr/level_badge donde quieras (perfil, dashboard, foro). El estilo vive en post.scss → .otr-level-badge.
